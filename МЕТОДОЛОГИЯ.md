# Методология инструмента «Генератор CSV для операторов связи»

Документ описывает, **как устроен и как работает** инструмент `index.html`.
К нему можно вернуться в любой момент, чтобы вспомнить логику или передать её другому человеку.

- **Репозиторий:** https://github.com/dshelkovenko-ui/csv-tool
- **Живой сайт (GitHub Pages):** https://dshelkovenko-ui.github.io/csv-tool/
- **Главный файл:** `index.html` (всё в одном файле: разметка, стили, логика)

---

## 1. Назначение

Инструмент превращает «сырой» список телефонных номеров в готовые CSV-файлы
по шаблону оператора связи (5 колонок). Используется, чтобы быстро подготовить
выгрузку устройств/номеров без ручного оформления таблиц.

---

## 2. Как это работает (важно для понимания ограничений)

- Это **статическая HTML-страница**. Сервера нет.
- Вся обработка (нормализация, удаление дублей, проверка, формирование CSV)
  происходит **в браузере пользователя**, на его устройстве.
- Введённые номера **никуда не отправляются и нигде не сохраняются** — это гарантия приватности.
- Следствие: одновременных пользователей фактически **без ограничений**; единственный
  «мягкий» лимит — трафик GitHub Pages (~100 ГБ/мес), что при размере страницы ~21 КБ
  соответствует ~5 млн загрузок в месяц.

---

## 3. Правила обработки номеров

### 3.1. Нормализация — итог всегда `7XXXXXXXXXX`
Каждый номер приводится к единому виду: **ровно 11 цифр, первая цифра — 7**.

| Что ввели | Условие | Результат |
|---|---|---|
| `8XXXXXXXXXX` | 11 цифр, начинается с 8 | `7XXXXXXXXXX` (8 → 7) |
| `+7XXXXXXXXXX` / `7XXXXXXXXXX` | 11 цифр, начинается с 7 | `7XXXXXXXXXX` |
| `9XXXXXXXXX` | 10 цифр, начинается с 9 | `79XXXXXXXXX` (добавляется 7) |

Перед проверкой из токена удаляются все нецифровые символы (`+`, `-`, `(`, `)`, пробелы).
Итог проверяется регулярным выражением `^7\d{10}$`.

### 3.2. Удаление дублей
- Дубли определяются **по нормализованному номеру**, а не по написанию.
  То есть `89001234567`, `+79001234567` и `9001234567` — это **один и тот же номер**.
- Лишние повторы удаляются автоматически (через `Set`), порядок сохраняется.
- При **вставке** (Ctrl+V) поле сразу пересобирается: остаются уникальные номера,
  показывается, сколько дублей удалено.

### 3.3. «Странные» номера (не проходят проверку)
Всё, что **не приводится** к `7XXXXXXXXXX`, считается странным и:
- **не попадает** в итоговые CSV-файлы;
- выводится отдельно с просьбой проверить формат.

Примеры странных номеров: `1234567` (мало цифр), `800123456789` (12 цифр),
`8900123456` (10 цифр с ведущей 8), `+7999111223` (10 цифр с ведущей 7).

---

## 4. Разбивка на файлы

- В один CSV-файл попадает **не более 1000 номеров** (`CHUNK_SIZE = 1000`).
- Список делится на блоки по 1000, **остаток** уходит отдельным последним файлом.
- Пример: **7432** номера → **8 файлов**: 7 файлов по 1000 + 1 файл с 432.
- Файлы скачиваются по очереди с задержкой 0,4 сек, чтобы браузер не блокировал
  множественную загрузку (при первом разе браузер может спросить разрешение —
  нужно нажать «Разрешить»).

### Имена файлов
- Один файл: `Оператор_N_номеров.csv`
- Несколько файлов: `Оператор_часть_K_из_M_N_номеров.csv`

---

## 5. Формат итогового CSV

- **Кодировка:** UTF-8 с BOM (`﻿`) — корректно открывается в Excel.
- **Разделитель:** точка с запятой `;`
- **Перенос строк:** `\r\n`
- **Колонки (5 штук):**
  1. Наименование устройства
  2. Идентификационный номер  *(= номер телефона)*
  3. Тип устройства
  4. Номер телефона
  5. Адрес, где находится устройство

Значения экранируются по правилам CSV (кавычки/спецсимволы оборачиваются в `"`).

---

## 6. Интерфейс

**Параметры файла:** оператор (Билайн / МТС / Мегафон / Теле2), тип устройства,
наименование (опц.), адрес (опц.).

**Список номеров:** большое поле ввода. Формат свободный — номера можно разделять
переносами строк, запятыми, точками с запятой, пробелами. Ограничений на количество
вставляемых номеров нет.

**Счётчик** под полем показывает в реальном времени:
`Номеров: N → файлов: M · дублей: D · на проверку: K`.

**Кнопки:**
- **Проверить** — не формирует файлы, а выдаёт комментарий:
  - ✓ «Всё корректно, можно формировать файлы» — если проблем нет;
  - ⚠ перечень замечаний (дубли + странные номера) — если проблемы есть;
  - ✗ если корректных номеров не найдено.
- **Сформировать и скачать CSV** — нормализует, убирает дубли, делит по 1000
  и скачивает файлы; странные номера в файлы не попадают и перечисляются в сообщении.
- **Очистить** — сбрасывает поля.

---

## 7. Как вносить изменения и публиковать

Репозиторий подключён по SSH, ветка по умолчанию — `main`.

```bash
cd "путь/к/csv_tool"
# ...правки в index.html...
git add index.html
git commit -m "описание изменения"
git push origin main
```

После пуша сайт на GitHub Pages обновляется автоматически за ~1 минуту.

## 8. Ключевые места в коде (`index.html`)

| Функция | Назначение |
|---|---|
| `classify(raw)` | делит ввод на `valid` (нормализованные) и `invalid` (странные) |
| `parseNumbers(raw)` | уникальные валидные номера (дедуп) |
| `fmtList(arr)` | форматирует список номеров для сообщений (с ограничением длины) |
| `updateCounter()` | живой счётчик под полем ввода |
| `checkNumbers()` | кнопка «Проверить» — формирует комментарий |
| `buildCSV(...)` | собирает текст CSV (заголовок + строки) |
| `chunkArray(arr, size)` | разбивка на блоки по 1000 |
| `downloadCSV(csv, name)` | скачивание одного файла |
| `generate()` | основной сценарий: проверка → разбивка → скачивание |
